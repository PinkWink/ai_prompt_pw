4. Threading Model (flask_yolo_app.py)

Multi-Thread Architecture
Inference Thread (daemon)
  • Hailo VDevice 초기화 + HEF 로드
  • inference_loop() — 무한 루프
  • OpenCV VideoCapture 프레임 읽기
  • cv2.resize() 640×640 전처리
  • pipeline.infer() Hailo NPU 추론
  • parse_nms_output() 결과 파싱
  • draw_detections() 오버레이
  • Lock 획득 → DetectionState 갱신
Flask Main Thread
  • app.run() HTTP 서버 (포트 5000)
  • GET / → HTML 페이지 반환
  • GET /video_feed → MJPEG 스트림 생성
  • GET /detections → JSON 감지 결과
  • GET /sources → 소스 목록 (MP4 스캔)
  • POST /set_source → 소스 전환 + restart_flag
  • generate_mjpeg() — State에서 프레임 읽어 전송
Browser Client (JavaScript)
  • <img src="/video_feed"> MJPEG 수신
  • setInterval(300ms) → /detections 폴링
  • FPS 배지, 감지 목록, 클래스 카운트 업데이트
  • loadSources() → 드롭다운 채우기
  • changeSource() → POST로 소스 변경 요청
DetectionState (Shared, Thread-Safe)
lock: threading.Lock
frame: np.ndarray
detections: list[dict]
fps: float
source: str
restart_flag: bool

5. REST API Endpoints

EndpointMethodResponseDescription
/ GET HTML 웹 UI 메인 페이지
/video_feed GET MJPEG Stream 실시간 영상 스트림 (bbox 오버레이 포함)
/detections GET JSON {fps, source, detections[{class_name, confidence, bbox}]}
/sources GET JSON 사용 가능한 비디오 소스 목록 (카메라 + MP4 파일)
/set_source POST JSON 비디오 소스 전환. Body: {"source": "camera"}
