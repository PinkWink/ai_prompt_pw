## 사용할 Confluence 매크로

  • ac:structured-macro ac:name="panel" : 색상 글상자 (개념 설명, 결과 해설 등)
  • ac:structured-macro ac:name="panel" (코드 설명용) : 코드 상세 설명도 펼침(expand)이 아닌 글상자(panel)로 표시
  • ac:structured-macro ac:name="info" : 페이지 상단 안내 문구
  • ac:structured-macro ac:name="tip" : 학습 포인트 / 팁

## 페이지 구성 패턴

  • 1. 상단에 H3 레벨로 "본 자료의 구성" 이라고 하고, 자료 전체의 흐름을 블록선도(flow diagram) 이미지로 시각화하여 삽입
    • Confluence에 draw.io 앱이 설치되어 있으므로, ac:structured-macro ac:name="drawio" 매크로를 사용하여 블록선도를 직접 페이지에 삽입할 것
    • 블록선도 스타일: 각 단계를 둥근 사각형(rounded rectangle) 블록으로 표현하고, 화살표(→)로 연결. 밝은 배경색과 진한 테두리를 사용하여 가독성 확보
    • draw.io 매크로 삽입이 불가능한 경우, Python(matplotlib/Pillow 등)으로 블록선도 이미지를 생성한 후 Confluence 페이지에 첨부파일로 업로드하여 ac:image로 삽입할 것
    • 텍스트로만 "A → B → C" 형태로 나열하는 것은 금지. 반드시 시각적 블록선도여야 함
  • 2. 핵심 개념을 panel 매크로로 정리 (밝은 배경 + 진한 타이틀바)
  • 3. 각 섹션별 이미지 아래에 panel 매크로(글상자)로 코드 설명 추가 (expand/펼침 사용 금지)
  • 4. 실행 결과 이미지 아래에 panel 매크로(타이틀 없음)로 결과 해설
  • 5. 마지막에 비교 정리표와 tip 매크로로 학습 포인트 정리
  • 6. 표(table)의 헤더 셀에는 해당 섹션 색상의 연한 톤을 background-color로 적용

## H3 제목과 슬라이드 모드

사용자는 Confluence의 슬라이드 모드를 H3 등급 기준으로 사용한다. 따라서:

  • 내용이 길어지거나 표와 같은 내용이 추가되면 적절한 위치에 H3(<h3>) 제목을 추가하여 슬라이드 단위를 나눠줄 것
  • 하나의 H3 섹션에 내용이 너무 많이 몰리지 않도록, 개념 설명/코드 설명/실행 결과 등을 각각 별도의 H3로 분리
  • H3 제목은 해당 섹션의 내용을 간결하게 요약하는 형태로 작성 (예: "각 도메인으로 rclpy 초기화", "실행 결과")
  • H1은 큰 파트 구분용, H3는 슬라이드 단위 구분용으로 사용
  • 글상자(panel)가 여러 개 연속으로 나오는 구간에는 반드시 H3 제목을 달아서 슬라이드 단위를 구분할 것
  • 모든 문장은 개조식으로 작성할 것
